# Casos de Teste — Cantina

Casos de teste detalhados em Gherkin, derivados das Condições de Teste (`4 - Condições de Teste.txt`) e dos Critérios de Aceitação Gherkin do README.

Cada caso segue o padrão:

> `**TCxx — Título** · Tipo · Camada · CTs cobertas`

- **Tipo**: Positivo · Negativo · Borda · Permissão.
- **Camada**: Ambas (padrão) · API · Web (apenas quando o comportamento diverge entre camadas).
- **CTs**: condições de teste exercitadas.

Mensagens de erro são citadas literalmente apenas quando carregam informação essencial ao usuário (números, limites, próxima ação). Nos demais casos, o "Então" descreve a categoria do erro em linguagem de negócio.

## US01 — Login de usuário

**TC01 — Cliente loga com credenciais válidas** · Positivo · Ambas · CT02

```gherkin
Cenário: Cliente autentica com sucesso
```

```gherkin
  Exemplos:
    | email             | senha    |
    |                   |          |
```

**TC03 — Login com senha incorreta** · Negativo · Ambas · CT03

